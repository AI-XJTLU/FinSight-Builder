# FinSight Builder Task2 Support

This folder stores real operation evidence for the generated Flask application after it has been run. It should not contain simulated screenshots or claims of successful deployment before those checks are actually executed.

## Evidence to replace manually after final commit, push, and deployment

The PNG files currently in `screenshots/` were captured before the final submission-readiness repair. Replace them with new real screenshots after the final commit has been pushed, Docker has been run again, and GitHub Actions has passed.

- `screenshots/01_commit_records.png`: GitHub commit history for version-control evidence.
- `screenshots/02_docker_deployed_website.png`: browser view of the Docker-served website showing `generated_market_banner.png`.
- `screenshots/03_github_actions_success.png`: successful GitHub Actions workflow run.
- `screenshots/04_api_health_or_stocks_optional.png`: optional but useful API JSON deployment evidence.
- `screenshots/05_docker_container_optional.png`: optional but useful Docker container evidence.
- `test_results/pytest_output.txt`: pytest output, only after tests are run.
- `test_results/local_runtime_check.txt`: concise endpoint checks from the Docker-served application.
- `test_results/docker_compose_ps.txt`: Docker Compose container state.

The official Software Component brief requires demonstration of deployment, version control, and CI/CD practices. It does not prescribe screenshot filenames. The filenames above keep the submitted evidence easy to inspect.
